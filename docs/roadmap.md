# BlazorThree roadmap

This roadmap is intended to show where contributions are most useful while the project is in preview.

## Near term

- Stabilize the preview API surface
- Improve documentation and task-oriented examples
- Expand demo coverage for common Blazor 3D use cases
- Tighten validation for animation, interaction, and model-loading scenarios
- Improve contributor ergonomics and CI feedback

## Candidate feature areas

- More end-to-end samples for product configurators, dashboards, and model viewers
- Additional geometry and material coverage where the underlying Three.js surface is already well understood
- Better asset-loading ergonomics and diagnostics
- More scene interaction examples and event-handling patterns
- Documentation for performance tradeoffs and recommended usage patterns

## Contribution priorities

The highest-value contributions right now are:

- Reproducible bug reports with a minimal demo
- Small, targeted fixes to rendering or interaction behavior
- Documentation additions that reduce first-run friction
- Examples that show real application scenarios
- CI and testing improvements

## Not a promise

This roadmap is directional, not a guarantee. Preview feedback and real-world usage should shape the order of work.
